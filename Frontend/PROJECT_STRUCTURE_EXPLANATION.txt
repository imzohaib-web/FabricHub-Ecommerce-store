FABRIC HUB - PROJECT FLOW & REACT CONCEPTS DOCUMENTATION
========================================================================

This document outlines the complete architectural flow, file-by-file organization, and React concepts used within the Fabric Hub e-commerce application.

------------------------------------------------------------------------
1. COMPLETE PROJECT FLOW
------------------------------------------------------------------------

The Fabric Hub application operates as a client-side single page application (SPA). Here is the step-by-step lifecycle of how it loads and executes:

1. Entry & Bootstrapping:
   - index.html is loaded by the browser and loads the entry point script: `src/main.jsx`.
   - `main.jsx` initializes React, mounts the App on the DOM root element, and wraps `<App />` in `<BrowserRouter>` to enable client-side routing.

2. Global Layout & Context Providers (src/App.jsx):
   - The `<App />` component acts as the global layout template.
   - It wraps all child elements in two global providers:
     * `<AuthProvider>` (handles user session and access control)
     * `<CartProvider>` (handles shopping cart state, totals, discounts, promo codes)
   - It defines a consistent visual structure: a sticky `<Navbar />` at the top, a dynamic `<main>` container in the middle, a footer at the bottom, and a persistent floating `<WhatsAppButton />` for instant customer support.

3. Page Routing (src/routes/AppRoutes.jsx):
   - The `<AppRoutes />` component maps specific URLs to page components:
     * `/` -> `Home.jsx` (Landing page with carousels, category grid, and featured highlights).
     * `/shop` -> `Shop.jsx` (Catalog with search, filtering by category, size, price, and sorting logic).
     * `/category/:slug` -> `Category.jsx` (Dynamic route showing products within a specific category, e.g., silk, lawn).
     * `/product/:id` -> `ProductDetails.jsx` (Detail page showing product details, interactive variant selection, reviews, and wishlist toggling).
     * `/cart` -> `Cart.jsx` (Checkout steps: Cart Summary -> Shipping Form -> Payment -> Success Screen).
     * `/login` & `/register` -> `Login.jsx` & `Register.jsx` (User authentication pages).
     * `/admin` -> `Admin.jsx` (Dashboard for managing inventory via CRUD operations and listing user orders. Restrained to Admin users only).
     * `*` -> Catch-all route that redirects back to `/`.

4. State Synchronisation with LocalStorage:
   - Authentication, wishlists, products lists, and cart items are stored and read from local storage using custom hooks and utils. This ensures that refreshing the browser does not lose user data (persistent shopping session).

3. REACT CONCEPTS USED IN THE PROJECT (EXPLAINED IN SIMPLE TERMS)
------------------------------------------------------------------------

Here are the React and modern web development concepts used in this project:

1. Components & Props
   - Components: Small, self-contained sections of code that return HTML-like code (JSX). For example, `Navbar.jsx` contains the logic and visual code specifically for the header.
   - Props (Properties): Arguments passed into components to make them dynamic and reusable, similar to parameters passed to a function.
     * Example: `<ProductCard product={item} />` allows the single card component to display details for different products depending on which item is passed.

2. State Management (`useState`)
   - State represents the local memory of a component. When a state variable changes, React automatically re-renders the component to show the updated information.
   - Example: The quantity of a product inside `QuantitySelector.jsx` is tracked using:
     `const [quantity, setQuantity] = useState(1);`
     When the user clicks '+', `setQuantity(quantity + 1)` runs, instantly changing the count shown in the UI.

3. Side Effects (`useEffect`)
   - Used to run instructions at specific points in a component's life, such as when it first loads (mounts) or when specific state values change.
   - Example: In `AuthContext.jsx`, a `useEffect` runs once when the app starts to check if a user session exists in local storage.

4. Global State Sharing (Context API: `createContext` & `useContext`)
   - In standard React, passing data down several layers of components (prop drilling) is tedious. Context solves this by allowing components to share global variables directly.
   - The application has two main context files:
     * `AuthContext.jsx`: Provides the `currentUser` object and `login`/`logout` status to components like the `Navbar` (to display username) and the `Admin` page (to block unauthorized access).
     * `CartContext.jsx`: Exposes the `cartItems` list, subtotal, and methods like `addToCart` to any page in the application.

5. Custom Hooks (Extracting Business Logic)
   - Custom hooks are custom JavaScript functions whose names start with `use`. They allow developer to extract stateful logic and reuse it across different components.
   - Examples:
     * `useLocalStorage`: Reads and writes values directly to browser storage automatically.
     * `useWishlist`: Keeps track of whether a product is liked or not and allows toggling it.

6. Client-Side Routing (`react-router-dom`)
   - Implements multi-page navigation without reloading the entire website.
   - Features:
     * `<Routes>` and `<Route>` specify the path and the element to display.
     * Dynamic parameter paths (`/product/:id` or `/category/:slug`) allow capturing values from the URL via the `useParams()` hook (e.g., loading product details using `id`).
     * `Navigate` and `useNavigate()` allow programmatic redirections (e.g., sending a user to home `/` when clicking logout or registering).

7. Lists and Keys (Rendering Arrays)
   - Used when displaying lists of items (like a grid of products). The code maps over the array and returns JSX elements.
   - React requires a unique `key` prop on each mapped element so it can track changes, insertions, or deletions efficiently.
   - Example:
     ```javascript
     {productsList.map((product) => (
       <ProductCard key={product.id} product={product} />
     ))}
     ```

8. Performance Optimization (`useMemo`)
   - Memoization cache results of expensive computations. The computation is only rerun when its specific inputs (dependencies) change.
   - In `Shop.jsx`, the operations of filtering and sorting products are wrapped in `useMemo` so they don't run on every single render (e.g. if the user toggles a unrelated modal), but only when search query, active category, or sorting order changes.

9. Conditional Rendering
   - Showing or hiding portions of UI dynamically using standard JS logic operators like `{condition && <Component />}` or ternary operators `{condition ? <YesComponent /> : <NoComponent />}`.
   - Example: In `Navbar.jsx`, if the user is authenticated, the profile dropdown is shown; otherwise, a "Login" button is rendered.

10. Animations & Transitions (Framer Motion)
    - Fabric Hub integrates `framer-motion` (utilizing `<motion.div>`, `<AnimatePresence>`) to provide modern animations like fade-ins, slide-ins, and hover scaling.
    - Example: Hovering over a product card triggers subtle scaling and shadow animations. Opening the cart drawer triggers a smooth sliding transition from the right edge.
